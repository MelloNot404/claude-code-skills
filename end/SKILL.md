# end

结束当前工作流，将本次会话的有效内容归档到本地 `worklog\` 目录下的 txt 文件，并按会话类型调整记录重点。

> 归档路径默认为当前项目下的 `worklog\`。用户可根据需要自行修改为个人工作目录。

归档只基于当前会话已经发生的内容，不为了归档继续执行任务、读取历史归档或补充排查。

## 调用方式

```
/end
```

## 执行流程

### 第一步：读取会话上下文

总结当前会话中已经实际发生的内容，包括：
- 本次会话的目标
- 已完成的事项
- 涉及的文件和 Git 操作
- 重要决策
- 未完成事项

### 第二步：判断会话类型

根据本次会话实际内容，判断属于以下哪一类或哪几类：

- **开发**：代码修改、调试、Git、项目操作
- **配置**：安装软件、环境变量、API 接入、工具配置
- **学习**：知识学习、概念理解
- **规划**：路线选择、任务安排、工作流决策
- **其他**：不属于以上分类的会话

### 第三步：确保归档目录存在

如果 `worklog\` 目录不存在，则创建该目录。

### 第四步：确定文件名

获取当前本地时间，文件名格式为：

```
yyyy-MM-dd-HH-mm.txt
```

如果同一分钟内文件名已存在，追加序号避免覆盖：

```
yyyy-MM-dd-HH-mm-02.txt
yyyy-MM-dd-HH-mm-03.txt
```

### 第五步：写入归档

根据会话类型调整记录重点（详见下文），将总结写入 txt 文件。

### 第六步：报告

写入成功后，只报告以下信息：

```
已归档
文件：worklog\yyyy-MM-dd-HH-mm.txt
会话类型：<开发/配置/学习/规划/其他>
遗留事项：<有/无>
```

完成后不要主动继续新任务，等待用户下一次输入。

## 归档内容格式

```
本次对话归档
时间：yyyy-MM-dd HH:mm
所在工作目录：<仅项目目录名，不含用户名的绝对路径>
会话类型：开发 / 配置 / 学习 / 规划 / 其他（可多选）

一、本次目标
- 一句话概括。

二、实际完成事项
- 按会话类型调整记录重点（详见下方说明）。

三、重要决策
- 工作流选择、技术选择或放弃事项。

四、未完成事项与风险
- 仍未完成的任务、待确认问题或风险。
- 如果没有，写"无"。

五、下次继续入口
- 最多 3 条下次可直接继续的动作。
```

## 根据会话类型调整记录重点

### 开发类
- 记录修改了哪些文件及相对路径
- 运行过什么验证命令及结果（测试/lint/构建）
- 是否执行了 Git commit / push，成功与否
- 遗留的代码问题或待测功能

### 配置类
- 记录配置目标和涉及的软件/工具
- 是否验证成功（运行状态、测试结果）
- 后续如何使用该配置
- **永远不记录密钥值**，涉及认证时只写"已配置相关认证信息，密钥未记录"

### 学习类
- 记录核心知识点和涉及的主题范围
- 主要卡点或理解难点
- 易错处或特殊注意事项
- 建议下次继续的内容和参考方向

### 规划类
- 记录最终决策和依据
- 被放弃的选项及原因
- 下一步具体动作和时间安排

### 其他
- 按实际内容简要记录即可

## 安全限制

- 绝不记录、展示或复制 API Key、token、password、secret、credential、.env 中的敏感值
- 涉及认证或 API 配置时，只写"已配置相关认证信息，密钥未记录"
- 不自动执行 git add、git commit、git push
- 不上传任何归档文件到 GitHub
- 不修改当前项目代码
- 除创建归档文件外，不进行额外文件操作
- 不读取本次会话无关的历史归档文件
- 不记录包含用户名的本机绝对路径；需要路径时优先使用项目名或相对路径

## 边界规则

- `/end` 的职责只有归档当前对话并结束当前工作流
- 归档时不要继续排查问题、继续修改代码或扩大任务范围
- 会话信息不足以确定某项是否完成时使用"待确认"，不要猜测
